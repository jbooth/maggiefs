from read perspective:
  look up inode, get data, scan blocks

from write perspective: 
  look up inode, acquire writelock, modify data or blocks

from link/mkdir perspective:
  lock parent, child, add link child entry on parent, linkedBy entry on child

from unlink perspective:
  lock parent, child, remove link from parent, child, if no parents, feed to incremental gc


from incremental gc perspective:
  pull from incremental queue, dealloc blocks
  

from full gc perspective:
  scan all inodes, delete if unreachable from root, check all blocks for proper replication/rebalancing
